# 执行监控功能快速入门

## 5分钟快速体验

### 1. 启动应用

```bash
# 确保已安装依赖
pip install -r requirements.txt

# 启动WebUI
python webui.py --port 7788
```

浏览器访问：http://localhost:7788

### 2. 配置Agent（首次使用）

#### 步骤1：进入Agent Settings标签页

点击顶部的"⚙️ Agent Settings"标签。

#### 步骤2：配置LLM

```
LLM Provider: 选择你的提供商（如：openai, zkh等）
LLM Model Name: 选择模型（如：gpt-4o, ep_20250805_urdq等）
LLM Temperature: 0.6（默认）
Use Vision: ✅ 勾选（启用视觉功能）
```

#### 步骤3：设置步数限制

```
Max Run Steps: 30（默认值，可根据任务复杂度调整）
```

**建议值**：
- 简单任务：10-20步
- 中等任务：20-50步
- 复杂任务：50-100步

#### 步骤4：配置API Key（如需要）

```
Base URL: 你的API端点（可选）
API Key: 你的API密钥（可选，如已在.env中配置则留空）
```

### 3. 运行第一个监控任务

#### 步骤1：进入Run Agent标签页

点击顶部的"🤖 Run Agent"标签。

#### 步骤2：观察实时指标面板

左侧会显示"📊 实时执行指标"卡片，包含：
```
状态: 等待任务
执行统计: 
- 当前步数: 0 / 0
- 总耗时: 0.00秒
Token消耗: 
- 总Token: 0
重试统计: 
- 总重试: 0
```

#### 步骤3：输入测试任务

在"Your Task or Response"输入框中输入：
```
打开百度首页，搜索"人工智能"，并截图
```

#### 步骤4：提交任务

点击"▶️ Submit Task"按钮。

#### 步骤5：观察实时更新

**指标面板会实时更新**：
```

执行统计: 
- 当前步数: 3 / 30
- 总耗时: 5.23秒
- 平均步骤耗时: 1.74秒

- Prompt Tokens: 1500
- Completion Tokens: 500
- 总Token: 2000

重试统计: 
- 系统级重试: 0

- 总重试: 1
```

**Chatbot区域会显示**：
- 每一步的执行详情

### 5. 测试步数熔断

#### 步骤1：设置较小的步数限制

返回"⚙️ Agent Settings"，将"Max Run Steps"设置为5。

#### 步骤2：提交复杂任务

输入一个需要多步骤的任务：
```
打开淘宝，搜索"笔记本电脑"，筛选价格5000-8000元，
按销量排序，查看前3个商品的详情
```

#### 步骤3：观察熔断效果

当执行到第5步时，系统会自动停止：
```

- 当前步数: 5 / 5  ← 达到上限
- 总耗时: 12.34秒

错误信息:

```

### 6. 分析重试情况

#### 系统级重试示例

当网络不稳定时，你会看到：
```
重试统计:
- 系统级重试: 3  ← 网络超时、连接错误等
- 业务级重试: 0
- 总重试: 3
```

#### 业务级重试示例

当页面加载慢或验证失败时：
```
重试统计:
- 系统级重试: 0
- 业务级重试: 2  ← 等待元素加载、输出验证失败等
- 总重试: 2
```

### 7. Token优化实践

#### 监控Token消耗

观察指标面板中的Token统计：
```
Token消耗:
- Prompt Tokens: 5000  ← 如果过高，考虑优化
- Completion Tokens: 1500
- 总Token: 6500
```

#### 优化建议

**如果Token消耗过高**：

1. **减少上下文长度**：
   - 在Agent Settings中降低"Max Input Tokens"
   - 从128000降到64000或32000

2. **优化任务描述**：
   - 使用更简洁明确的描述
   - 避免冗余信息

3. **关闭Vision（如不需要）**：
   - 取消勾选"Use Vision"
   - 可以显著减少Token消耗

4. **使用更小的模型**：
   - 从gpt-4o切换到gpt-4o-mini
   - 或使用本地模型（ollama）

### 9. 故障排查

#### 问题1：任务未完成就停止了

**症状**：
```
状态: STEP_LIMIT_EXCEEDED
当前步数: 30 / 30
```

**解决方案**：
1. 增加"Max Run Steps"值
2. 简化任务描述
3. 检查是否有重复步骤

#### 问题2：Token消耗异常高

**症状**：
```
Token消耗:
- 总Token: 50000+  ← 异常高
```

**解决方案**：
1. 检查是否有循环调用
2. 关闭Vision功能
3. 减少Max Input Tokens
4. 优化系统提示词

#### 问题3：重试次数过多

**症状**：
```
重试统计:
- 系统级重试: 10+  ← 异常多
```

**解决方案**：
1. 检查网络连接
2. 验证API配置
3. 增加超时时间
4. 检查目标网站是否可访问

### 10. 高级功能

#### 导出执行历史

任务完成后，可以下载：
- **Agent History JSON**：完整的执行历史数据
- **Task Recording GIF**：任务执行过程的动画

#### 暂停和恢复

在任务执行过程中：
- 点击"⏸️ Pause"暂停执行
- 点击"▶️ Resume"继续执行
- 点击"⏹️ Stop"停止任务

#### 清除历史

点击"🗑️ Clear"按钮清除：
- 聊天历史
- 浏览器会话
- 执行状态

### 11. 最佳实践

#### ✅ 推荐做法

1. **合理设置步数限制**
   - 根据任务复杂度设置
   - 留有余量（+20%）

2. **监控Token消耗**
   - 定期检查Token统计
   - 优化高消耗任务

3. **分析重试原因**
   - 系统级重试 → 检查基础设施
   - 业务级重试 → 优化任务逻辑

4. **保存执行历史**
   - 下载JSON文件
   - 用于后续分析和优化

#### ❌ 避免做法

1. **步数限制过小**
   - 会导致任务无法完成
   - 建议至少20步

2. **忽略Token消耗**
   - 可能导致高额费用
   - 定期检查和优化

3. **不分析重试**
   - 错过优化机会
   - 可能掩盖潜在问题

## 快速参考

### 关键指标说明

| 指标 | 说明 | 正常范围 |
|------|------|----------|
| 当前步数 | 已执行的步骤数 | < max_steps |
| 总耗时 | 任务总执行时间 | 取决于任务 |
| 平均步骤耗时 | 每步平均时间 | 1-5秒 |
| Prompt Tokens | 输入Token数 | 取决于任务 |
| Completion Tokens | 输出Token数 | 取决于任务 |
| 系统级重试 | 系统异常重试 | 0-2次 |
| 业务级重试 | 业务逻辑重试 | 0-3次 |

### 执行状态说明

| 状态 | 说明 |
|------|------|
| RUNNING | 正在执行 |
| SUCCESS | 成功完成 |
| FAILED | 执行失败 |
| STEP_LIMIT_EXCEEDED | 超过步数限制 |
| CANCELLED | 用户取消 |

### 快捷键

|------|--------|
| 提交任务 | Enter（在输入框中） |
| 暂停/恢复 | 点击按钮 |
| 停止 | 点击按钮 |
| 清除 | 点击按钮 |
